Новое в этой версии (после 2026-05-07)

Сделано параллельно по плану, до отправки вам на повторный тест.

«выдай доступ baeру X» — бот сразу создаёт buyer'а в Vibero. Опечатка → лишний buyer, ручная чистка.
Mutation Confirm Flow: «Создам байера 'X' с auto credentials. Подтверди: «да» / «отмена»». Опечатки больше не идут в production.
Бот не отвечает на процедурные вопросы вне 12 keyword-тем («как настроить робота автовыгрузки?», «как добавить статус брокера?»)
LLM Wiki по Karpathy pattern: 38 страниц doc.vibero.tech скомпилированы на Opus, бот выбирает 1-3 релевантные → синтезирует ответ с цитатой doc_url.
Один сигнал (LLM confidence) — бот может «крутиться» в одной теме 5+ сообщений до ручной эскалации
Composite escalation signals: confidence + repetition + ALL CAPS + profanity + frustration phrases. Score≥2 → автоэскалация с reasons в логе оператора.
ChatHistory in-memory — терялся при рестарте, клиенты получали «бот забыл что мы обсуждали»
JSON persistence (только redacted text), flush каждые 5 сообщений. После рестарта бот помнит последние 12 реплик per chat.
Vision OCR прицеплен в код, но не валидирован end-to-end
Smoke-тест на скриншоте deals_list прошёл — извлекает «Showing 1-50 of 168 items», IDs, статусы. Можете прислать скрин в чат — бот распарсит и пропустит через pipeline.

Что в работе

Категории, где бот пока эскалирует или отвечает не оптимально. План есть, очерёдность согласовываем.

Глубокий troubleshooting (под клиента)
«Лиды не приходят, раньше работало» → сейчас бот эскалирует. На пилоте хочется автодиагностика 4 типовых причин: флаг Send via API, капы, whitelist, reject_reason из /lead/view. Требует знания конкретного сетапа клиента — поэтому делаем под одного на пилоте.

Real screenshots для Vision
Pipeline (Telegram → OCR → dispatcher) подключён и smoke-протестирован. Чтобы покрыть реальные форматы (карточки лидов, ошибки админки, JSON-payload'ы) — пришлите 5-10 скриншотов из ваших чатов.
Жду материалов от вас
Multi-tenant config
Сейчас бот привязан к demo.vibero.net. Чтобы подключать реальные клиенты (Leadovich, Masonic, ваш subdomain) — нужен per-chat routing: chat_id → поддомен → buyer_id/template. Проектно сделано, активируем при выборе пилотного клиента.
Перед пилотом
CSAT и метрики качества
Inline-кнопки 👍/👎 после каждого ответа. Считать repeat-contact rate (вернулся ли клиент с тем же intent за 24h) — это правильный resolution rate, не deflection. Дашборд в Langfuse.
На пилоте
Cost monitoring + token budget
Hard cap per-conversation (например 8K input + 2K output) → если превышен, эскалация. Алерт на дневной spike. На demo не критично, на пилоте 50 чатов могут стоить $300/день вместо $50.
На пилоте
Redis вместо JSON для persistence
Сейчас ChatHistory сохраняется в JSON-файл (после рестарта помнит контекст). На пилоте с multi-worker нужен Redis: shared idempotency-кэш + TTL для ChatHistory. Тривиальный апгрейд — drop-in adapter.

Как удобно протестировать

Чем больше реальных формулировок — тем точнее картина. Не подыгрывайте боту, пишите как обычно.

  1. Откройте чат VIBERO INTERNAL DEMO в Telegram. Бот @vebero_demobot.
  2. Прогоните типовой день: создание офиса → языки/гео → тест-лид → кап → freeze. Можно из памяти, можно подсмотреть в свежих чатах с реальными клиентами.
  3. Если ответ мимо — просто перешлите эту реплику Максу с комментарием «вот тут хотел X, бот ответил Y». Каждое сообщение логируется, разберёмся быстро.
  4. Команды: /start — disclosure, /human — явная эскалация, /version — текущий код, /whoami — chat_id.
