Что это и кому подходит

Обычный запуск Директа — это связка «копирайтер + дизайнер + директолог», 2–3 недели и 150–300к₽. Здесь всё делает один человек, отдавая рутину AI-агенту: сбор семантики, анализ конкурентов, тексты, заливку кампаний, ежедневную аналитику и рекомендации. На человеке остаётся три вещи — куда идём, что из предложенного принимаем, и проверка, что агент не наврал.

Подходит, если

  • Нужно быстро проверить спрос и unit-экономику по цифрам
  • Нет бюджета на агентство / штат маркетинга
  • Готов сам держать руль (стратегия и решения — за тобой)
  • Хочешь прозрачность: видеть каждое действие и его эффект

Что получаешь на выходе

  • Кампании, собранные и залитые через API
  • Дашборд трафика: воронка по источникам, CAC, рекомендации
  • Лог всех действий (что, когда, зачем, какой эффект)
  • Утренний дайджест в Telegram с рекомендациями
Как устроено

Подключение — один раз

Что настроить, чтобы всё заработало

Разовый сетап на ~1–2 часа. Дальше только работаешь.

1

Claude Code + подписка

Терминальный AI-агент от Anthropic. Ставится как CLI. Нужна подписка (Max-план окупается сразу). Понимает русский, работает со скиллами и файлами проекта.

2

Установить 5 Яндекс-скиллов

Четыре готовых скилла (wordstat, metrika, webmaster, search-api) — базово с GitHub Полякова, MIT-лицензия. Скилл yandex-direct — свой. Весь набор, собранный и готовый к установке, лежит в открытом репозитории:

→ github.com/mkultraaaa/claude-yandex-skills — Claude Code skills для Direct, Metrika, Webmaster, Wordstat, Search API

Каждый скилл — папка с SKILL.md (плейбук для агента), scripts/ (обёртки над API) и config/.env (токен). Клонируешь в ~/.claude/skills/ — и агент их видит.

git clone https://github.com/mkultraaaa/claude-yandex-skills
# скопировать папки скиллов в ~/.claude/skills/
3

Создать OAuth-приложение в Яндексе и получить токены

Один раз регистрируешь приложение на oauth.yandex.ru с доступами к Директу, Метрике, Wordstat, Вебмастеру. Получаешь ClientID и по нему — OAuth-токены. Их кладёшь в config/.env каждого скилла (YANDEX_DIRECT_TOKEN, YANDEX_METRIKA_TOKEN и т.д.).

# config/.env скилла yandex-direct
YANDEX_DIRECT_TOKEN=y0_AgAAAA...
# API: https://api.direct.yandex.com/json/v5/  (Bearer token)
4

Проверить, что API живой

Один вызов — и видно все кампании аккаунта. Дальше агент оперирует ими по имени/ID.

cd ~/.claude/skills/yandex-direct
bash scripts/campaigns.sh            # список кампаний
bash scripts/balance.sh              # баланс и расход
bash scripts/stats.sh --date1 ... --date2 ...   # статистика
5

Chrome в debug-режиме (для UI-хаков)

Для операций, которых нет в API. Запускаешь Chrome с отладочным портом и отдельным профилем (без него порт игнорируется), логинишься в Директ руками один раз — дальше Playwright работает с этой сессией.

open -a "Google Chrome" --args \
  --remote-debugging-port=9222 \
  --user-data-dir=/tmp/chrome-debug-profile
Подключение — подробно

OAuth-приложение Яндекса: где и что кликать

Это самый неочевидный шаг. Один раз регистрируешь приложение, выдаёшь ему доступы к нужным API и получаешь токен, который кладёшь в скиллы. Ниже — точный маршрут по кликам.

⚙️ Что делаем по сути

OAuth-приложение — это «пропуск», через который твой агент ходит в API Яндекса от твоего имени. Регистрируешь приложение → выбираешь доступы (Директ, Метрика, Вебмастер) → получаешь ClientID → по нему добываешь токен → токен в config/.env.

Часть A. Зарегистрировать приложение

1

Открыть oauth.yandex.ru, войти под аккаунтом, где у тебя Директ и Метрика. Нажать «Создать приложение».

2

Название — любое понятное, напр. MyDirect CLI.

3

Платформа → выбрать «Веб-сервисы». В поле Redirect URI вписать служебный адрес Яндекса, который просто показывает токен:

https://oauth.yandex.ru/verification_code
4

Доступы (Data / права) — добавить, поставив галки в нужных сервисах:

СервисЧто выбрать
Яндекс.Директ«Использование API Яндекс.Директа» (direct:api)
Яндекс.Метрика«Получение статистики» + при необходимости «Управление» (metrika:read / metrika:write)
Яндекс.Вебмастер«Просмотр и управление сайтами» (webmaster:hostinfo, webmaster:verify)
Wordstatидёт вместе с доступом к Директу — отдельно не нужен
5

Создать. На странице приложения появятся ClientID и Client secret — они понадобятся дальше. ClientID можно копировать сразу.

⚠️ Отдельно про доступ к Директ API

Для боевого Директа доступ к API у аккаунта обычно открыт сразу. Если API отвечает ошибкой прав — в интерфейсе Директа зайди в Настройки → раздел для разработчиков / API и убедись, что доступ к API включён (для новых рекламодателей иногда нужна заявка, подтверждают быстро).

Часть B. Получить токен

6

Открыть в браузере ссылку, подставив свой ClientID (response_type=token — это «выдай токен сразу»):

https://oauth.yandex.ru/authorize?response_type=token&client_id=ВАШ_CLIENT_ID
7

Яндекс попросит войти и нажать «Разрешить». После этого перекинет на страницу, где в адресе (или на самой странице) будет access_token=y0_Ag... — это твой токен. Скопировать.

8

Вписать токен в config/.env каждого скилла, которому он нужен:

# ~/.claude/skills/yandex-direct/config/.env
YANDEX_DIRECT_TOKEN=y0_AgAAAA...
# тот же токен подходит для metrika/webmaster/wordstat, если доступы выданы
9

Проверка боем — один вызов, и видно все кампании аккаунта:

cd ~/.claude/skills/yandex-direct
bash scripts/campaigns.sh

Если вернулся список кампаний (или пусто, но без ошибки авторизации) — подключение работает.

✅ Проверено на живом аккаунте

По этой связке агент не только читает статистику, но и создаёт кампании: черновик заливается через API и появляется в кабинете за секунды (создание → проверка → удаление проходят end-to-end). На демо это и показываем: ставим задачу на русском → агент собирает и заливает кампанию → она возникает в интерфейсе Директа.

Рабочий процесс

Как запускается кампания — 8 шагов

Наш реальный маршрут от «есть продукт» до «реклама крутится». Каждый шаг — задача агенту на русском.

1

Семантика через Wordstat

Агент собирает ядро скиллом yandex-wordstat: кластеры B2C/B2B, частотности, правая колонка, минус-слова. У нас: 450+ фраз, 20 кластеров за 2 часа. Вывод сразу читается — где массовый спрос, где перехват брендов.

2

Анализ конкурентов через SERP

yandex-search-api — ТОП-10 по приоритетным запросам. Кто в топе, какие офферы, где дыры в выдаче. 8 запросов — 30 минут. Решает, по чему бить в Директе, а что отдать в SEO.

3

Портреты ЦА через AI

3 персоны за 30 минут — гипотезы, которые потом прибиваются цифрами. Из них: на какие боли бить, как разводить B2C/B2B-треки, что показать на лендинге первым.

4

Лендинг через контент-пайплайн

6 этапов: ресёрч → критерии → драфт → экспертное ревью → тест на персонах → аудит. Персоны как фидбэк-машина ловят возражения до трат на рекламу (у нас — цена картинки и юрлицо).

5

Структура кампаний + заливка через API

Агент собирает структуру (кампании → группы → ключи → объявления → UTM) и заливает скиллом yandex-direct одним проходом. У нас — 4 кампании, 8 групп, 61 ключ, 12 объявлений за 8 минут в статусе DRAFT.

6

Ревью директолога — 9 правил

Один разовый звонок (40 мин, ~5к₽) → 9 правил, каждое стоит бюджета, если не знать (см. ниже). Правила зашиваются в память проекта — агент подтягивает их при любой работе с Директом.

7

Chrome CDP — добить UI-настройки

Автотаргетинг (оставить «целевые»), снять товарную галерею, задать ценность цели. Playwright проходит все группы за 15 минут вместо 4–6 часов ручного клика.

8

Запуск → мониторинг → оптимизация

Модерация ~1 день. Дальше — ежедневный отчёт через yandex-metrika + дашборд + утренний дайджест в Telegram с рекомендациями (поднять бюджет / выключить / поднять ставки). Каждое изменение → в лог действий.

Знание, которое стоит денег

9 правил директолога — как не слить бюджет

То, что мы получили за один платный звонок. Каждое правило, если его не знать, съедает бюджет.

01. Товарную галерею на поиске — отключать для сервисных/SaaS-продуктов. Карусель товаров бьёт мимо интента, конверсия нулевая.

02. Автотаргетинг — только «Целевые» + «Без упоминания бренда». «Широкие» = чистый слив (искал борщ — увидел твою рекламу).

03. Ставки не ниже 5–7₽ на старте. За копейки Яндекс просто не откручивает объявления.

04. Быстрые ссылки и уточнения — обязательны. 4 + 4. Без них падает рейтинг качества → клик дороже, позиция ниже.

05. UTM — только динамические. {campaign_id}, {ad_id}, {keyword}. Статические UTM = потеря данных.

06. Отображаемые ссылки — осмысленные. ng-ai.ru/chatgpt вместо длинного URL с параметрами. CTR +10–15%.

07. Ценность вовлечённой сессии ~50₽. Оптимизируй под ценность микро-конверсии, а не под «цену клиента» — иначе показов слишком мало.

08. Минимальный дневной бюджет — 300₽/кампанию (лимит Яндекса). 4 кампании = минимум 1 200₽/день.

09. Сравнительная реклама — мимо модерации. «Лучше ChatGPT» отклонят. «ChatGPT из России без VPN» — можно.

Контур управления

Дашборд, лог действий и рекомендации

Это то, что отличает «покликать в кабинете» от системы: каждое действие фиксируется, каждое утро приходит срез с рекомендациями, а дашборд показывает воронку по источникам.

📊 Дашборд трафика

Локальный веб-интерфейс, тянет Метрику через API.

  • Воронка по источникам: визит → рег. начал → завершил → пополнение
  • CAC и CR по каждой кампании
  • Авто-рекомендации (_build_insights): что масштабировать, что резать
  • Отдельные срезы: трафик, аудитория, ключи, финансы, Вебмастер

📝 Лог действий — actions.jsonl

Агент пишет запись после каждого изменения кампаний.

{"ts":"2026-05-16T10:48",
 "type":"pause",
 "target":"NG_MatchCam_v1",
 "what":"Остановлены — дорого, CR 0.6%",
 "details":"165 кликов, ~1700₽, 0 оплат",
 "by":"max"}

📬 Утренний дайджест в Telegram (9:00)

Скрипт подтягивает вчерашнюю статистику и присылает срез: показы, клики, CTR, воронка регистраций, CAC по кампаниям — и готовые рекомендации («Direct_Match: поднять бюджет 300→600₽», «PPP: выключить — CAC 327₽ втрое дороже среднего»). Решение принимаешь за 30 секунд с телефона.

💡 Главный урок: бюджет сливается на форме, а не в рекламе

Воронка: 575 кликов → 463 визита → 267 открыли форму → 39 завершили. Между кликом и регистрацией теряется 93%. Мы полезли чинить не ставки, а форму: добавили оффер «+100₽ на баланс» и сократили поля. Результат — доходимость +85%, CAC по Direct_Match упал 199→87₽. Один абзац текста дал кратный эффект. Реклама была не виновата — виновата была дверь, в которую никто не мог войти.

🎯 Правильный порядок приоритетов

До запуска зашей цели в аналитику (иначе Директ оптимизируется по клику, а не по конверсии). Не плоди лишние кампании — 2–3 трека (прямой спрос + перехват + опц. B2B) хватает. Оффер на форму — с первого дня. И считай unit-экономику до запуска, а не когда деньги уже улетели.

Честно

Чего этот подход НЕ заменяет

  • Стратегию и решения. Агент исполняет, но «куда идём» и «что принимаем» — на человеке. Проверять, что не наврал, обязательно.
  • Ревью директолога. AI не знает нюансов кабинета уровня «галерея сливает бюджет». Разовый звонок специалиста окупается кратно.
  • Живые интервью с ЦА. AI-персоны — прицел на 30 минут, а не истина. У нас 70% гипотез подтвердились цифрами, но это валидация, не замена ресёрча.
  • Разработку на 20% операций. Скилл под Директ и Chrome CDP требуют рук/кода. Без них 80% всё равно делается, но не «под ключ».
