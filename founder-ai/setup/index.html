Claude Code для собственника: от чистого Mac до рабочей системы

Логика системы: приложение — руки, файлы — мозг. Всё, что ассистент знает о человеке и его бизнесе, лежит в обычных папках. Поэтому настройка — это 4 переключателя в приложении и macOS плюс раскладка правильных файлов. Дальше — по шагам, галочки кликаются.

Приложение Claude Code (залогинен, подписка)

│

    └── personal/         ← личное

Поведение в приложении

Три вещи, которые снимают страх и дают контроль.

  1. Разрешения. Claude спрашивает перед правкой файлов и командами. Правки в ~/projects — разрешать («Allow always»). Незнакомая команда — спросить в чате: «объясни по-простому, что сделает эта команда». Это не формальность, а контроль над системой
  2. Терминал не нужен. Всё через чат. Исключение — если Claude сам попросит нажать Enter в терминале справа
  3. Результат не тот — не начинать заново, а сказать: «получилось не то, я хотел X». Переделка — нормальный цикл, не ошибка

Правила задач — зашиты в личный CLAUDE.md
  1. Просьба неоднозначна → ассистент сначала озвучивает, как понял
  2. Большая задача → план и «готово, когда X»
  3. Делать только то, что просили
  4. Две неудачи подряд → стоп и разбор, не третья попытка вслепую
  5. «Сделал» = показал доказательство, а не «должно работать»
6~60 мин

Первый час: сценарий по ценности

Порядок принципиальный: сначала польза на его данных, объяснения по ходу. Не наоборот.

  1. Дашборд за 10 минут. Его выгрузка из МИС/CRM → перетащить в clinic/dashboard/data/ → «собери дашборд по этим данным» → открылась страница с его цифрами. Момент «понял»
  2. Инсайты. «Какие 3 вещи в этих цифрах заслуживают внимания?» — разговор о бизнесе, не о технологии
  3. Знакомство по ходу. Теперь Claude уточняет контекст (кто, цель года, услуги) — и клиент видит, зачем отвечает: каждый ответ ложится в CLAUDE.md, ответы ассистента становятся точнее на глазах
  4. Первая боль. Одна реальная задача из его списка — от формулировки до результата
  5. Ритуал. Показать START.md: «завтра утром — 15 минут, вот первый вопрос»
7

Чего не делать

  • Персональные данные пациентов (ФИО, телефоны, диагнозы) ассистенту не давать — выгрузки обезличивать
  • Не подтверждать удаление, если непонятно, что удаляется
  • Не вести всё в одной сессии неделями — контекст «протухает»; новая тема = новая сессия
  • Не пересказывать таблицы словами — файл в inbox/, потом вопрос
